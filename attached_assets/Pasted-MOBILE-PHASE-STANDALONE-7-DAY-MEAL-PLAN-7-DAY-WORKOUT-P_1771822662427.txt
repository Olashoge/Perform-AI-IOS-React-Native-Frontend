2) Standalone Meal Plan Wizard (7-day)
--------------------------------------------
Create: app/meal/new.tsx

Requirements:
- Block creation if profile is missing (route to Profile).
- Show a “Using your profile” summary card at top with Edit Profile link.
- Prefill all relevant fields from profile (but allow overrides; do not modify profile).

Form Fields (align with web meal plan form):
- Primary goal (default from profile goal if present)
- Meals per day (2 or 3)
  - If 2 meals: allow selecting which two (Breakfast/Lunch/Dinner)
- Dietary style (multi-select if supported by backend)
- Cuisine style (multi-select)
- Allergies & Intolerances (prefill)
- Foods to avoid (prefill)
- Favorite meals (prefill)
- Spice preference (if exists)
- Prep style (Meal Prep, Quick, etc. if exists)
- Pace (optional) with everyday helper text

Start Date:
- Include a start date picker on this page or as final step.
- Conflict-aware: block dates that would overlap existing scheduled MEAL plans (same logic as web for meal plans).
- After creation, plan should be scheduled if user selected a start date; otherwise unscheduled.

Generate Flow:
- On submit, call the existing backend endpoint to generate a standalone meal plan.
- Show generating screen with staged progress:
  - Building your meal plan
  - Compiling your grocery list
  - Estimating prices (if applicable)
  - Finalizing
- On success: navigate to Meal Plan detail page (existing plan detail route for meal plan).
- Ensure plan appears in Plans Hub under Meal Plans.
